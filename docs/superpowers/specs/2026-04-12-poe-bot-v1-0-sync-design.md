# POE Bot Pure v1.0 Sync Design

**Goal**

将 `poe_bots/microcap_top100_poe_bot.py` 收敛为一个仅服务 POE 云端的纯 `v1.0` 独立机器人，并把当前主线 `microcap_top100_mom16_biweekly_live.py` 中已经生效的 `v1.0` 刷新、成本净值续算和默认查询口径同步进单文件实现。

**Context**

- 终端脚本体系负责本地 `1.0 / 1.1 / 1.2` 策略运行与查询。
- POE 机器人在 POE 云端独立运行，不能依赖仓库主线脚本或本地 CSV 作为运行前提。
- 当前 POE 机器人仍保留 `1.1 / 1.2` 多版本解析与文案，且 `v1.0` 行为已与主线最近更新存在漂移。

**Approved Scope**

- POE 机器人改成纯 `1.0`。
- 删除 POE 里的 `1.1 / 1.2` 版本识别、策略切换和相关提示文案。
- 同步主线 `v1.0` 的近期关键规则：
  - 默认查询口径固定为 `v1.0`
  - 刷新时保留 `proxy_turnover` 历史
  - 可基于 `proxy_turnover` 重建成本净值
  - 非调仓尾部允许无 turnover 历史的成本净值续算
  - 使用刷新元数据校验执行时序/研究栈版本

**Non-Goals**

- 不把 POE 机器人改成依赖本地主线脚本的包装器。
- 不继续维护 POE 内的 `1.1 / 1.2` 查询能力。
- 不重构终端策略脚本体系。

**Architecture**

- 保持 `poe_bots/microcap_top100_poe_bot.py` 单文件部署形态。
- 在单文件内部新增一组“mainline-compatible v1.0”辅助函数，承接主线 `v1.0` 新增的刷新与成本逻辑。
- 命令入口仍由 POE 脚本负责，但命令帮助、表现输出和缓存命名全部收敛到纯 `v1.0` 语义。

**Component Changes**

- 命令层：
  - 删除版本识别与策略切换状态。
  - 帮助文案、介绍文案、附件命名统一按纯 `v1.0` 输出。
- 表现层：
  - 只加载 `v1.0` 的表现数据源。
  - 删除多版本标签、固定对冲比切换和 `1.1/1.2` fallback。
- 刷新/成本层：
  - 新增 `proxy_turnover` 路径与序列化支持。
  - 新增 `rebuild_costed_nav_from_proxy_turnover`。
  - 新增 `try_extend_costed_nav_without_turnover`。
  - 将元数据版本检查对齐主线 `v1.0`。
- 测试层：
  - 增加 POE 纯 `v1.0` 行为测试。
  - 增加成本净值重建/续算测试。

**Error Handling**

- 缺少 `proxy_turnover` 且无法走非调仓尾部续算时，明确报错，不静默回退到旧逻辑。
- 发现旧的版本化查询文本时，不再切换版本；机器人始终按 `v1.0` 返回。
- 表现数据缺失时，继续使用现有的清晰错误提示，但只针对 `v1.0` 数据源。

**Testing**

- 验证版本解析被移除后，命令仍正确落到 `信号/实时信号/成分股/进出名单/表现`。
- 验证帮助文案和介绍文案不再出现 `1.1 / 1.2`。
- 验证 `v1.0` 成本净值可由 turnover 历史重建。
- 验证非调仓尾部可在无 turnover 历史时续算成本净值。

**Risks**

- POE 单文件体量较大，局部改动容易漏掉旧的版本化分支。
- 主线和 POE 单文件并非完全同构，实现同步时需要用行为对齐而不是逐行复制。
- 表现数据在 POE 云端可能优先走内嵌数据源，需要避免与旧多版本逻辑残留耦合。
