# v5.8 策略组合改动说明文档

## 概述

v5.8 是对 v5.4 策略组合的全面优化版本，汇总了 v5.5~v5.8 四个版本的所有改动。本次 v5.7→v5.8 的改动为：**移除 Sub-A 和 Sub-A-DK 的冷却期参数(COOLDOWN_DAYS)**。

### 子策略组成（权重不变）
| 子策略 | 权重 | 说明 | 最后修改版本 |
|--------|------|------|------------|
| Sub-A | 15% | A股乖离动量 + R²过滤 + 国债ETF | v5.5(重构) → v5.8(移除冷却期) |
| Sub-A-DK | 15% | 多配对Top-1 + 二值R² + VolScaling | v5.6(重构) → v5.8(移除冷却期) |
| Sub-B | 40% | 美股9ETF动量轮动 + VolReg | v5.7(参数优化) |
| Sub-C | 30% | 美股生产组合 + Vol-Scaling | v5.4(不变) |

## 一、v5.8 改动：移除冷却期

### 1.1 改动内容

| 子策略 | 原值 | 新值 | 改动 |
|--------|------|------|------|
| Sub-A | COOLDOWN_DAYS = 3 | 移除 | 删除参数和冷却逻辑 |
| Sub-A-DK | COOLDOWN_DAYS = 5 | 移除 | 删除参数，信号直接赋值 |

### 1.2 审查依据

#### Sharpe稳定性扫描

CD=0~3 形成小平台（得分0.51~0.57），差异微小。

**Sub-A-DK (测试范围 CD=0~15):**

| CD | 4yr Shrp | 8yr Shrp | 12yr Shrp | 16yr Shrp | 稳定性得分 |

#### 交易频率分析（决定性证据）

**Sub-A:**

| CD | 年均交易次数 | vs CD=0 差异 |
|----|-------------|-------------|
| 0 | 29.2 | 基准 |
| 1 | 29.2 | 0.0 |
| 3 | 27.9 | -1.3 |

CD=0 到 CD=3 仅减少 1.3 次/年交易，冷却期几乎不起作用。

**Sub-A-DK:**

| CD | 年均交易 | 方向翻转 | 配对切换 |
|----|---------|---------|---------|
| 0 | 33.4 | 3.3次/年 | 30.1次/年 |
| 5 | 33.3 | 3.3次/年 | 30.1次/年 |
| 15 | 34.1 | 4.1次/年 | 30.1次/年 |

**关键发现：年均33次交易中30次是配对切换，仅3次是方向翻转。冷却期只控制方向翻转，对配对切换无效。因此 COOLDOWN_DAYS 对 Sub-A-DK 完全无效。**

高CD值反而增加交易次数（阻止方向翻转 → 信号漂移到其他配对 → 额外配对切换）。

### 1.3 移除原因总结

1. **乖离动量信号本身足够平滑**：slope(price/MA60, 20天) 是双重平滑操作，信号不会频繁翻转
2. **R²过滤是天然冷却**：趋势不清时自动停手，比固定天数更智能
3. **T+1制度已天然保证最少1天间隔**：CD=0 和 CD=1 结果完全相同
4. **Sub-A-DK 的冷却期完全无效**：交易由配对切换主导（30/33次），方向翻转极罕见（3次/年）

### 1.4 代码改动

**BiasMotionConfig (Sub-A):**
- 删除 `COOLDOWN_DAYS = 3`

**BiasMotionStrategy.run() (Sub-A):**
- 删除 `last_trade_day = -999` 变量
- 原来的冷却判断：
  ```python
  if ideal != holding and (i - last_trade_day) >= cfg.COOLDOWN_DAYS:
      target = ideal
      last_trade_day = i
  ```
- 简化为：
  ```python
  target = ideal if ideal != holding else None
  ```

**MultiPairDKConfig (Sub-A-DK):**
- 删除 `COOLDOWN_DAYS = 5`

**_run_single_pair():**
- 原来的30行冷却循环：
  ```python
  sig = [np.nan] * n
  cur = np.nan
  last_t = -999
  for i in range(n):
      ds = d['daily_signal'].iloc[i]
      if pd.isna(ds) or i < start_idx:
          sig[i] = cur; continue
      if pd.isna(cur):
          cur = ds; last_t = i
      elif ds != cur and (i - last_t) >= cfg.COOLDOWN_DAYS:
          cur = ds; last_t = i
      sig[i] = cur
  d['signal'] = sig
  ```
- 简化为3行直接赋值：
  ```python
  d['signal'] = np.nan
  valid = d['bias_mom'].notna() & (np.arange(n) >= start_idx)
  d.loc[valid, 'signal'] = np.where(d.loc[valid, 'bias_mom'] > 0, 1, -1)
  d['signal'] = d['signal'].ffill()
  ```

## 二、历史改动汇总（v5.4 → v5.8 全部改动）

### v5.4 → v5.5：Sub-A 重构
- 将 MA-Turning 轮动替换为乖离动量 + R²过滤 + 国债ETF
- 资产池：A500, 创业板, 沪深300, 中证1000, 中证500, 国债ETF
- 参数：BIAS_N=60, MOM_DAY=20, R2_WINDOW=20, R2_THRESHOLD=0.3

### v5.5 → v5.6：Sub-A-DK 重构
- 从单配对(SZ50/ZZ1000)改为多配对Top-1
- 5指数 → C(5,2)=10配对，每天选信号最强的1对
- 新增二值R²过滤：R²(20)<0.3 → 次日收益置零
- 波动率缩放：TARGET_VOL=0.15, VOL_WINDOW=30, MAX_LEV=1.5

### v5.6 → v5.7：Sub-B 参数优化
- US_ROT_LB：120 → 160（120在平台边缘，160更稳健）
- US_ROT_REBALANCE_THRESHOLD：1.3 → 1.0（移除，全时段贡献为负）
- 其余不变：MIN_TURNOVER=0.15, VolReg=2.0(10/250), TOP_N=3

### v5.7 → v5.8：移除冷却期
- Sub-A COOLDOWN_DAYS=3 → 移除
- Sub-A-DK COOLDOWN_DAYS=5 → 移除

### Sub-C 审查结论
- 全部参数（TARGET_VOL, VOL_WINDOW, MAX_LEV, MIN_LEV, THRESHOLD, TIMING）均在最优位置
- Vol-Scaling 全时段正贡献（+0.04~0.09 Sharpe），确认保留
- 择时机制有害（8yr/12yr Sharpe下降），确认关闭
- **无需变更**
