# 策略组合 v5.4 → v5.5 变动说明

## 一、变动背景

v5.4 组合包含四个子策略：

| 子策略 | 说明 | 权重 |
|--------|------|------|
| Sub-A | A股5ETF轮动 (MA-Turning) | 15% |
| Sub-A-DK | 中证1000/上证50 多空 | 15% |
| Sub-B | 美股9ETF轮动 + VolReg | 40% |
| Sub-C | 美股生产组合 + Vol-Scaling | 30% |

在对 Sub-A 进行过拟合检验时，发现严重问题，因此对 Sub-A 进行了彻底改造。**其他三个子策略 (Sub-A-DK / Sub-B / Sub-C) 和组合权重均未改动。**

---

## 二、Sub-A 过拟合问题

### 2.1 v5.4 原始 Sub-A 的设计

v5.4 的 Sub-A (`run_cn_strategy`) 使用以下逻辑：
- **选股信号**：MA20 turning point（均线拐点）
- **过滤模式**：`CN_MA_FILTER_MODE = "turning"` — 均线由跌转升时入场
- **标的池**：5只A股权益ETF（中证红利、创业板、沪深300、中证1000、中证500）
- **无避险资产**：空仓时收益为0（无国债轮动）

### 2.2 发现的过拟合证据

对 Sub-A 进行前后半段测试和参数敏感性分析时，发现以下问题：

1. **前后半段收益差异过大**：策略表现严重依赖特定市场环境（如2014-15牛市），前半段远好于后半段，说明信号可能对历史特定行情过度拟合
2. **MA-Turning 参数敏感**：`MA_WINDOW=20` 的选择缺乏足够鲁棒性，微调参数会导致结果大幅变化
3. **缺少趋势质量验证**：仅靠均线拐点判断入场时机，无法区分有效趋势和随机波动

### 2.3 结论

Sub-A 的 MA-Turning 信号过拟合风险较高，需要从选股逻辑层面进行彻底改造，而非简单调参。

## 三、v5.5 新 Sub-A 设计

### 3.1 核心思路：乖离动量 + R²过滤 + 国债ETF

新 Sub-A 采用完全不同的选股框架：

```
选股信号 = 乖离动量 = slope( price/MA60 归一化后最近20日 ) × 10000
趋势过滤 = 滚动R²(20) >= 0.3
避险资产 = 国债ETF (511010), 不受R²过滤
```

### 3.2 乖离动量 (Bias Momentum) 指标

**计算步骤：**

1. **偏离度 (Bias)**：`bias = price / MA(60)` — 价格相对60日均线的偏离程度
2. **取窗口**：取最近20天的 bias 序列
3. **归一化**：`bias_norm = bias / bias[0]` — 消除不同标的的绝对水平差异
4. **拟合斜率**：对 bias_norm 做线性回归，取斜率 × 10000

**设计意图：**

- MA60 做长期趋势锚定，比 MA20 更稳定，不容易被短期噪声干扰
- 归一化后拟合斜率，衡量的是"偏离度变化的加速度"而非绝对位置
- 选乖离动量最高的标的 = 选趋势加速最快的品种

### 3.3 R² 趋势质量过滤

**解决的问题：** 在震荡市中，乖离动量可能产生虚假信号（价格随机波动也会产生非零斜率）。

**机制：**
- 对权益ETF计算20日滚动R²（价格对时间的线性回归拟合优度）
- R² >= 0.3：趋势有效，允许持仓
- R² < 0.3：趋势不明确，持有现金（年化3%无风险收益）
- **国债ETF不受R²过滤**：国债走势平稳，R²经常不达标但仍有配置价值

### 3.4 国债ETF 加入轮动池

**v5.4 的缺陷：** 原 Sub-A 仅在5只权益ETF中轮动，空仓时收益为0。

**v5.5 的改进：** 加入国债ETF (511010) 作为第6个标的：
- 当国债的乖离动量最高时（通常出现在权益市场下跌期），自动切换到国债
- 国债不受R²过滤，提供了天然的避险通道
- 效果：减少空仓时间，在熊市中获取债券正收益

### 3.5 参数总结

| 参数 | 值 | 说明 |
|------|-----|------|
| BIAS_N | 60 | 均线周期 |
| MOM_DAY | 20 | 斜率拟合窗口 |
| R2_WINDOW | 20 | R²滚动窗口 |
| R2_THRESHOLD | 0.3 | R²最低门槛 |
| COOLDOWN_DAYS | 3 | 换仓冷却期 |
| COMMISSION | 0.1% | 单边佣金 |
| RF_ANNUAL | 3% | 现金无风险收益 |

### 3.6 标的池

| 代码 | 名称 | 类型 | R²过滤 |
|------|------|------|--------|
| 1.515100 | 中证红利ETF | 权益 | 是 |
| 0.159915 | 创业板ETF | 权益 | 是 |
| 1.000300 | 沪深300 | 权益 | 是 |
| 1.000852 | 中证1000 | 权益 | 是 |
| 1.000905 | 中证500 | 权益 | 是 |
| 1.511010 | 国债ETF | 债券 | **否** |

以下结果使用 v5.4 脚本的 `_fetch_data()` 从API获取数据（与 v5.4 bot 完全一致的数据源），仅替换 Sub-A 部分。Sub-A-DK、Sub-B、Sub-C 完全使用 v5.4 的策略函数，确保对比的公正性。

## 六、代码变更摘要

### 变更的文件

| 文件 | 变更 |
|------|------|
| `strategy_portfolio_v5.5.py` | 新增：完整的 v5.5 组合脚本，包含 BiasMotionStrategy 类和对比逻辑 |

### 不变的部分

- v5.4 主脚本 (`strategy_signal_v5.4_aggressive.py`)：不改动
- Sub-A-DK (`run_dk_strategy`)：不改动
- Sub-B (`run_us_rotation` + VolReg)：不改动
- Sub-C (`simulate_prod_btc_phased` + Vol-Scaling)：不改动
- 组合权重 `COMBINED_WEIGHTS`：不改动 (15% / 15% / 40% / 30%)

### 集成方式

v5.5 通过以下方式集成到 v5.4 框架：

1. **数据获取**：完全使用 v5.4 的 `_fetch_data()` 从 API 获取数据，确保数据一致性
2. **额外数据**：通过 v5.4 的 `fetch_cn_kline("1.511010")` 获取国债ETF
3. **策略替换**：用 `BiasMotionStrategy.run()` 替代 `run_cn_strategy()`
4. **指标计算**：完全照搬 v5.4 的 `_handle_performance` 逻辑（月度收益→对齐→组合NAV→日度覆盖MaxDD）

### 如需将新Sub-A整合到v5.4 bot中

在 v5.4 脚本的 `_run_strategies` 方法中，将：

```python
cn_result = run_cn_strategy(cn_close, CN_STOCK_CODES)
```

替换为：

```python
# 需要额外传入 cn_close_with_bond (包含国债ETF列的DataFrame)
from strategy_portfolio_v5_5 import BiasMotionStrategy
cn_result = BiasMotionStrategy().run(cn_close_with_bond)
```

同时在 `_fetch_data` 中增加获取国债ETF数据并拼入 `cn_close` 的逻辑。
