# Sub-A-DK 策略改造说明文档 (v5.4 → v5.6)

## 1. 改造背景

### 1.1 原版问题
v5.4 的 Sub-A-DK 策略存在 **参数脆弱性** 问题：
- 原版仅使用单配对 (上证50 / 中证1000)
- 核心参数 `MOM_LB=20` 处于性能悬崖边缘 — 微调即导致大幅性能变化
- 策略在不同回测区间表现差异较大，稳健性不足

### 1.2 改造目标
- 提升参数稳健性，消除对单一参数值的敏感依赖
- 扩大信号来源，降低单配对集中风险
- 保持策略简洁性，避免过度拟合

---

## 2. 改造方案

### 2.1 多配对扩展
从原版的 **1个配对** 扩展到 **10个配对**：

| 序号 | 配对 | 数据源 |
|------|------|--------|
| 1 | SZ50 / HS300 | cn_dk_close['DK_SZ50'] / cn_close['1.000300'] |
| 2 | SZ50 / ZZ500 | cn_dk_close['DK_SZ50'] / cn_close['1.000905'] |
| 3 | SZ50 / ZZ1000 | cn_dk_close['DK_SZ50'] / cn_dk_close['DK_ZZ1000'] |
| 4 | SZ50 / CYB | cn_dk_close['DK_SZ50'] / cn_close['0.159915'] |
| 5 | HS300 / ZZ500 | cn_close['1.000300'] / cn_close['1.000905'] |
| 6 | HS300 / ZZ1000 | cn_close['1.000300'] / cn_dk_close['DK_ZZ1000'] |
| 7 | HS300 / CYB | cn_close['1.000300'] / cn_close['0.159915'] |
| 8 | ZZ500 / ZZ1000 | cn_close['1.000905'] / cn_dk_close['DK_ZZ1000'] |
| 9 | ZZ500 / CYB | cn_close['1.000905'] / cn_close['0.159915'] |
| 10 | ZZ1000 / CYB | cn_dk_close['DK_ZZ1000'] / cn_close['0.159915'] |

5个指数：上证50(SZ50)、沪深300(HS300)、中证500(ZZ500)、中证1000(ZZ1000)、创业板(CYB)
C(5,2) = 10 配对

> **注意**: 只有 SZ50/ZZ1000 配对在2010年11月之前有数据。其余9个配对自2010年11月22日(创业板ETF上市)起可用。策略会自动处理这种数据不对齐，早期仅从可用配对中选择。

### 2.2 Top-1 选择
每天从所有可用配对中，选择 **|bias_mom| 信号最强的1对**：
- 使用前一日信号选择（避免前瞻偏差）
- 等权持仓该配对
- 信号缺失时不参与

### 2.3 二值R²过滤
对每个配对的价差累计曲线计算滚动R²：
- `R²(20日窗口) < 0.3` → **次日收益置零**（回避趋势不明确的市场）
- 二值方式：通过/不通过，不作为权重

### 2.4 其他策略组件（与v5.4一致）
每个配对内部保留完整的策略流程：
- **乖离动量信号**: `slope(price_ratio / MA(60), 归一化后最近20日拟合) × 10000`
- **波动率缩放**: 目标波动率15%, 30日窗口, 杠杆0.1~1.5
- **冷却期**: 5天方向切换冷却
- **交易成本**: 单边0.1%（翻仓4×, 初始建仓2×, scale变动2×）

## 3. 测试过程与备选方案

### 3.1 测试的方案总览

| 方案 | 描述 | 结论 |
|------|------|------|
| **Top-1 纯信号** | 每天选|bias_mom|最强的1对 | 收益好但回撤偏高 |
| **Top-3 等权** | 每天选前3对等权 | 分散过度，收益显著下降 |
| **等权10对** | 所有配对等权 | 收益最低 |
| **R² 权重** | R²值作为收益权重（R²越高权重越大） | 与二值R²接近，多一层复杂度无额外收益 |
| **二值R²** | R²<0.3 置零 | ✅ **最终选定** — 简洁有效，回撤改善明显 |
| **NAV_MA 过滤** | 净值<MA(N)时停仓 | MA130~300有宽平原，但增加复杂度，选择不加 |
| **二值R² + NAV_MA** | 两者叠加 | 表现最好但过度优化风险 |

### 3.2 Top-N 选择测试结果

| 方案 | 近8年年化 | 近8年夏普 | 近8年MDD |
|------|-----------|-----------|----------|
| Top-1 | 最高 | 最高 | 中等 |
| Top-3 | 明显下降 | 下降 | 略低 |
| 等权 | 最低 | 最低 | 最低 |

**结论**: Top-1 集中选择最强信号，在该策略逻辑下表现最优。

### 3.3 R²模式对比

| R²模式 | 近8年年化 | 近8年夏普 | 近8年MDD |
|--------|-----------|-----------|----------|
| 二值R²(R²<0.3→置零) | 较高 | 较高 | 较低 |
| R²权重(R²越高权重越大) | 接近 | 接近 | 接近 |

**结论**: 两种R²方式表现近似，选择更简洁的二值R²。

### 3.4 NAV_MA 平原检验

对二值R² + NAV_MA过滤进行了精细扫描 (MA20~300, 步长10)：
- **90%夏普平原**: 多个时段在MA130~300区间内夏普保持在最佳值的90%以上
- **80%夏普平原**: 更宽，覆盖MA80~300
- **各时段交集**: 近4/8/10年的≥80%夏普MA值有较大交集

**最终决策**: 虽然NAV_MA平原够宽够稳，但用户选择 **不加NAV_MA**，保持策略简洁。纯二值R²已经提供足够好的表现。

## 4. 最终参数

### 4.1 MultiPairDKConfig

| 参数 | 值 | 说明 |
|------|-----|------|
| `BIAS_N` | 60 | 均线周期（价格比的MA窗口） |
| `MOM_DAY` | 20 | 斜率拟合窗口（乖离动量计算窗口） |
| `R2_MODE` | `'binary'` | 二值R²过滤模式 |
| `R2_WINDOW` | 20 | R²滚动计算窗口 |
| `R2_THRESHOLD` | 0.3 | R²阈值（低于此值次日置零） |
| `COOLDOWN_DAYS` | 5 | 方向翻转冷却期 |
| `TARGET_VOL` | 0.15 | 目标年化波动率 |
| `VOL_WINDOW` | 30 | 已实现波动率计算窗口 |
| `MAX_LEV` | 1.5 | 最大杠杆倍数 |
| `MIN_LEV` | 0.1 | 最小杠杆倍数 |
| `SCALE_THRESHOLD` | 0.10 | scale变动阈值（低于此不调整） |
| `COMMISSION` | 0.001 | 单边佣金率 |
| `TOP_N` | 1 | 每日选择的配对数 |

### 4.2 与v5.4原版DK参数对比

| 参数 | v5.4 原版 | v5.6 新版 | 变化 |
|------|-----------|-----------|------|
| 配对数 | 1 (SZ50/ZZ1000) | 10 (C(5,2)) | **核心变化** |
| 选择方式 | 固定单配对 | Top-1动态选择 | **核心变化** |
| R²过滤 | 无 | 二值R²(0.3) | **新增** |
| 信号计算 | 乖离动量 | 乖离动量（同） | 不变 |
| 波动率缩放 | 有 | 有（同参数） | 不变 |
| 冷却期 | 5天 | 5天 | 不变 |
| 交易成本 | 0.1% | 0.1% | 不变 |

## 7. 改造总结

### 关键改进
1. **多配对扩展** (1→10对): 降低单配对集中风险，提升信号多样性
2. **Top-1动态选择**: 每天跟踪最强配对，最大化信号利用
3. **二值R²过滤**: 简洁有效的趋势确认机制，降低无趋势时期损耗
4. **自包含脚本**: v5.6脚本不依赖外部dk_*.py模块，所有逻辑内嵌

### 设计原则
- **简洁优先**: 在多个表现接近的方案中选择最简洁的（二值R²优于R²权重，不加NAV_MA）
- **稳健性**: 多配对天然降低对单一参数的敏感度
- **避免过拟合**: 拒绝在回测表现更好但增加复杂度的NAV_MA叠加方案
