# V5.8 更新日志：DK策略改用价格指数

## 更新背景

ADK策略在实际操作中使用**股指期货**和**ETF期权**，这些衍生品的盈亏跟踪的是**价格指数**而非全收益指数。因此DK策略的回测数据源应使用价格指数，以更准确地反映实际交易表现。

Sub-A 轮动策略仍使用全收益指数（包含分红再投资），因为Sub-A假设持有指数ETF并享受分红。

## 修改内容

### 1. DK策略代码改回价格指数

| 指数 | 旧代码(全收益) | 新代码(价格指数) |
|:-----|:--------------|:---------------|
| 中证1000 | H00852 / 1.H00852 | **000852** / 1.000852 |
| 上证50 | H00016 / 1.H00016 | **000016** / 1.000016 |
| 沪深300 | *(从cn_close读H00300)* | **000300** / 1.000300 |
| 中证500 | *(从cn_close读H00905)* | **000905** / 1.000905 |
| 创业板 | *(从cn_close读399606全收益)* | **399006** / 0.399006 |

### 2. cn_dk_close 扩展为5列

DK策略之前只有2列独立数据（ZZ1000、SZ50），其余3个（HS300、ZZ500、CYB）从cn_close读取（混用了全收益指数）。

现在 `cn_dk_close` 包含全部5列价格指数数据，DK策略完全独立于cn_close：

```
CN_DK_COLS = ["DK_ZZ1000", "DK_SZ50", "DK_HS300", "DK_ZZ500", "DK_CYB"]
```

`CN_DK_INDICES` 中所有条目的 `src` 均改为 `'dk'`（从cn_dk_close读取）。

### 3. CN_STOCK_CODES 移除沪深300

沪深300 (1.H00300) 之前只是为了给DK策略从cn_close读取而额外获取的。现在DK独立获取价格指数数据，不再需要：

```python
# 旧: CN_STOCK_CODES = CN_EQUITY_CODES + ["1.H00300"]
# 新: CN_STOCK_CODES = CN_EQUITY_CODES
```

### 4. DK数据获取增加Sina备用源

DK价格指数的获取源从 csindex + EastMoney 增加了 Sina 作为第三备用源：

```
csindex → EastMoney → Sina
```

### 5. 清理死代码

移除了 `hs300_col = "1.H00300"` 占位变量（v5.8无MA过滤，该变量未被使用）。

## 未修改部分

- **Sub-A 轮动策略**：仍使用全收益指数（H20955、399606、H00016、H00852、H00905）
- **国债指数**：仍使用 H11077 全收益指数
- **ZZHL拼接**：H00922 → H20955 逻辑不变
- **记录机器人**：持仓映射表 `_CN_HOLDING_NORM` 无需修改
- **美股策略(Sub-B/Sub-C)**：无影响

```
Sub-A (轮动策略)
  └─ cn_close: 全收益指数 (H20955, 399606, H00016, H00852, H00905)

```
